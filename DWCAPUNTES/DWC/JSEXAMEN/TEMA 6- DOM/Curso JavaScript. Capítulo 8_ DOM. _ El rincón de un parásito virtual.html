Curso JavaScript. Capítulo 8: DOM.

12 julio, 2010 Deja un comentario Go to comments

En este nuevo capítulo del curso de JavaScript dedicado a DOM (Document Object Model) vamos a meternos de lleno en la manipulación de las páginas web. Como sabéis, JavaScript es un lenguaje que se ejecuta en el navegador del usuario y se utiliza principalmente para dar dinamismo a nuestras páginas, comunicarse con el servidor mediante AJAX y crear interfaces ricas de usuario.

Las nuevas aplicaciones de la WEB 2.0 utilizan mucho código JavaScript para manipular todos los elementos de las páginas. Todas estas tareas son muy sencillas gracias a DOM.

DOM

DOM es una API de funciones que se utilizan para manipular archivos XML y por extensión permite manipular archivos XHTML. De ahí la importancia de crear páginas XHTML bien formadas y validas siguiendo los estándares del W3C. DOM no es exclusivo de JavaScript, la mayoría de lenguajes incoporan DOM para procesar archivos XML, como Java o PHP.

Para poder utilizar las funciones de manipulación de elementos, DOM tiene que transformar internamente el archivo XHTML original en una estructura más fácil de manipular. Esta estructura es una estructura en forma de árbol de nodos, donde cada elemento es un nodo del árbol.

Si JavaScript es el primer lenguaje de programación que utilizáis puede que os cueste un poco entender el concepto de árbol de nodos, así que vamos a ver un ejemplo sencillo de un documento XHTML y de árbol de nodos, extraído del libro de JavaScript de Javier Eguíluz.

<html xmlns="http://www.w3.org/1999/xhtml">
  <head>
    <meta http-equiv="Content-Type" content="text/html; charset=iso-8859-1" />
    <title>Pag ina sencilla</title>
  </head>

  <body>
    <p>Esta pagina es <strong>muy sencilla</strong></p>
  </body>
</html>

Utilizando las funciones de DOM podemos añadir, eliminar y modificar cualquier nodo de la página, ya sea para añadirle nuevos atributos, nuevos estilos CSS o algún que otro manejador de eventos. Lo malo de todo esto es que las especificaciones de DOM que realiza cada navegador incluye algunas diferencias que nos complican las cosas para crear aplicaciones que se ejecuten bien en diferentes navegadores. Por eso nacieron algunos frameworks JavaScript, como Prototype o jQuery, que proporcionan una solución cross-browser.

Tipos de nodos

Antes de entrar a ver la interfaz Node de JavaScript y las funciones de la API de DOM tenemos que explicar los principales tipos de nodos. Para entender los principales tipos de nodos, sería recomendable tener algo de idea de lo que es XML y saber definir documentos XML válidos y bien formados.

– Document: Es el nodo raíz de todos los documentos XHTML/XML.
– DocumentType: Es el nodo que contiene la representación del DTD empleado en la página (indicado mediante el DOCTYPE).
– Element: Representa el contenido definido por un par de etiquetas de apertura y cierre (<etiqueta>…</etiqueta>) o de una etiqueta abreviada que se abre y se cierra a la vez (<etiqueta/>). Es el único nodo que puede tener tanto nodos hijos como atributos. La mayoría de etiquetas XHTML se transforman en nodos de este tipo.
– Attr: Representa un atributo de una etiqueta.
– Text: Almacena el contenido del texto que se encuentra entre una etiqueta de apertura y una de cierre. También almacena el contenido de una sección de tipo CDATA.
– CDataSection: Es el nodo que representa una sección de tipo <![CDATA[ ]]>.
– Comment: Representa un comentario de XML.

Vamos a ver un ejemplo de archivo XML y su árbol con los diferentes tipos de nodos, extraído como no, del magnifíco libro escrito por Javier Eguíluz, uno de mis dioses, gracias él empezé a interesarme por el desarrollo de aplicaciones webs.

<?xml version="1.0"?>
<clientes>
  <!-- El primer cliente -->
  <cliente>
    <nombre>Empresa SA</nombre>
    <sector>Tecnologia</sector>
    <notas><![CDATA[
    Llamar la proxima semana
    ]]></notas>
  </cliente>
</clientes>

La interfaz Node

En JavaScript cada nodo del árbol es un objeto Node con una serie de constantes, propiedades y métodos que permiten manipular cada nodo.

Vamos a ver como utilizar algunos de estos métodos para acceder a los nodos de un documento, primero a través de sus nodos padres descendiendo jerarquicamente hasta encontrar el que queremos manipular y después accediendo al nodo directamente.

El documento de prueba que vamos a utilizar es el siguiente:

<html>
<head>
  <title>DOM</title>
</head>
<body>
  <p id="primero" name="parrafo">Primer parrafo</p>
  <p id="segundo" style="border-top-style:solid">Segundo parrafo</p>
  <p id="tercero" class="parrafo">Tercer parrafo</p>
</body>
</html>

Acceso jerárquico a los nodos

Empezamos obteniendo el nodo raíz del árbol, el nodo Document. Javascript dispone del objeto document que forma parte del BOM que veremos más adelante, este objeto se considera equivalente al nodo Document del documento:

var html = document;

Ahora podemos hacer uso de los métodos de la clase Node para acceder a sus hijos nodos a través de las propiedades firstChild y lastChild que devuelven el primer nodo hijo y el último nodo hijo respectivamente. Como sabemos que el nodo html solo cuenta con dos nodos hijos podemos obtener el nodo head obteniendo el primer nodo hijo y el nodo body obteniendo el último nodo hijo:

var head = html.firstChild;

var body = html.lastChild;

Aunque también podemos utilizar la propiedad childNodes que es un array que contiene los nodos hijos:

var head = html.childNodes[0];

var body = html.childNodes[1];

El objeto Node cuenta también con una serie de propiedades para acceder al nodo hermano anterior y al nodo hermano posterior, así como a su nodo padre y al documento al que pertenece.

var html = head.parentNode;

var body = head.nextSibling;

var head = body.previousSibling;

var document = head.ownerDocument;

El objeto Node también cuenta con un método para comprobar si un nodo tiene nodos hijos:

if (body.hasChildNodes())
{
  var parrafo1 = body.firstChild;
}

Acceso directo a los nodos

Los métodos que hemos visto hasta ahora requieren que vayamos recorriendo el árbol hasta el elemento que queramos manipular. En el ejemplo anterior para manipular el primer parrafo del documento tendríamos que haber accedido al nodo raíz del elemento, después descender al nodo body y por último descender al nodo párrafo. Imaginaros ahora que estamos en un documento bastante complejo con cientos de nodos de todos los tipos. Si queremos manipular uno de los últimos nodos (lo que en la jerga de la programación se conoce como hoja, dado que no contiene elementos hijos) tendríamos que recorrer cientos de elementos. Por este motivo no es eficiente acceder a los nodos de forma jerárquica sino directamente.

El objeto Node contiene una serie de métodos para acceder directamete a cualquier nodo del árbol de nodos del DOM.

– getElementsByTagName():

El método getElementsByTagName() obtiene una lista de todos los nodos del documento cuya etiqueta sea igual a la que se pasa como argumento.

var parrafos = document.getElementsByTagName('p');

Este método devuelve un array con todos los párrafos del documento. Realmente no devuelve un array normal, al igual que la propiedad childNodes, devuelve un objeto NodeList que podemos tratar como un array dado que como ya hemos dicho los objetos en JavaScript se comportan como arrays.

De esta forma podemos acceder al primer párrafo de la página accediendo al primer elemento:

var parrafo1 = parrafos[0];
– getElementsByName():

El método getElementsByName() obtiene una lista de todos los nodos del documento cuyo atributo name sea igual al que se pasa como argumento.

var parrafo = document.getElementsByName('parrafo');

De esta forma recuperamos el párrafo que tiene un atributo name=”parrafo”. De normal el atributo name suele ser único, pero por ejemplo, en los campos radiobutton el atributo name es común por lo que la función devolvería una lista.

– getElementById():

Por último, el método más utilizado, getElementById() que devuelve el nodo del documento cuyo atributo id sea igual al pasado como argumento.

var parrafo3 = document.getElementById('tercero');

Acceso a los atributos de los nodos

Hasta ahora hemos accedido a los nodos del documento pero no hemos realizado nada con estos nodos. Lo normal cuando accedemos a un nodo elemento es recuperar o modificar los valores de sus atributos. Podemos acceder a los atributos del nodo de dos maneras.

– Utilizando la propiedad attributes del objeto Node que devuelve un objeto NamedNodeMap que puede ser tratado como un array y que dispone de una serie de métodos para tratar los atributos.

– Y la forma que vamos a ver, muchisímo más sencillo que la anterior. Los atributos de los nodos se transforman automáticamente en propiedades de los nodos. Así que podemos utilizar la notación de puntos para acceder a los atributos. Solo hay que tener en cuenta que el atributo class cambia su nombre por className dado que class es una palabra reservada de JavaScript para su uso en un futuro. Además los nombres de las propiedades CSS que contengan guiones se eliminan y se utiliza la notación camelCase. Vamos a ver una serie de ejemplos para entender todo esto mejor:

var parrafo1 = getElementById('primero');
var parrafo2 = getElementById('segundo');
var parrafo3 = getElementById('tercero');

var atributoClass = parrafo3.className;
var atributoBorder = parrafo2.borderTopStyle;

Creando nodos:

Podemos crear nuevos nodos de dos maneras distintas, utilizando los métodos del objeto Node que proporciona DOM o utilizar la propiedad innerHTML. Hay mucha controversia en el mundo de los desarrolladores de que forma se debería implementar la creación de nuevos nodos, dado que innerHTML es una propiedad privada de Microsoft que por su exito fue incorporada en todos los navegadores, pero no está reflejada en el estandar DOM.

– Utilizando los métodos de DOM para crear nuevos nodos deberíamos seguir los siguientes pasos:

1. Creamos un nodo de tipo Element que represente el elemento que queremos crear.
2. Creamos un nodo de tipo Text que represente el contenido del elemento.
3. Establecemos los atributos que queramos que posea el elemento al nodo de tipo Element.
4. Añadimos el nodo Text como nodo hijo del nodo Element.
5. Añadimos el nodo Element como nodo hijo al nodo que esté en el lugar que le corresponde.

Vamos a ver un ejemplo sencillo para insertar un parrafo que sea hijo del elemento body:

var parrafo = document.createElement('p');
var contenido= document.createTextNode('Nuevo parrafo');
parrafo.setAttribute('id', 'nuevo');
parrafo.appendChild(contenido);
document.body.appendChild(parrafo);

El método appendChild() inserta el nuevo nodo al final del nodo padre. Si queremos insertar un nodo antes que otro o despues podemos utilizar los métodos insertBefore() e insertAfter().

– Como el proceso de crear e insertar nuevos nodos puede llegar a ser muy tedioso, Microsoft decidió ponerselo más fácil a los programadores e inventarse una nueva propiedad para insertar HTML puro directamente a un nodo. Esta manera es mucho más rápida pero no forma parte del estandar de DOM y tiene el inconveniente de que se inserta como una cadena en un nodo.

Vamos a ver el ejemplo anterior utilizando esta propiedad:

document.body.innerHTML='<p id="nuevo">Nuevo parrafo</p>';

Eliminando nodos

Para eliminar un nodo del árbol DOM podemos utilizar el método del objeto Node, removeChild(). Este método toma como argumento el objeto Node que se quiere eliminar.

var parrafo = document.getElementById('tercero');
parrafo.parentNode.removeChild(parrafo);

Modificando nodos

Podemos modificar el contenido de un nodo a través de la propiedad nodeValue o a través de la propiedad innerHTML.

alert(document.body.firstChild.nodeValue);
document.body.firstChild.nodeValue = 'Parrafo modificado';
alert(document.body.firstChild.nodeValue);

RESUMEN

El DOM permite manipular todos los elementos de un documento XHTML bien formado. De esta manera podemos crear todo tipo del dinamismo que tanto se utiliza en la WEB 2.0. Hemos visto una pequeña introducción a los métodos de la API de DOM pero son métodos tan potentes que te permiten hacer todo lo que se te pase por la cabeza con los elementos de una página, desde crear nuevos, borrar, insertar, modificar propiedades CSS, en fin de todo. El único inconveniente es que los navegadores realizan la implementación del DOM de forma distinta, por lo que algunos métodos o propiedades no funcionan igual.

Menos mal que contamos con unos cuantos frameworks escritos en JavaScript, como Prototype y jQuery, que nos permiten centrarnos en lo que queremos hacer ya que ellos mismos se encargan de que todo funcione igual en los navegadores a los que dan soporte, proporcionando una solución cross-browser.
